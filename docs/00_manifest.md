# Documentation Manifest

This package is split into short markdown files.
Each file should stay below 80 lines.

Core folders:
- `docs/`: project theory, rules, metrics, benchmark plan.
- `research/`: literature review and research procedures.
- `implementation/`: module and pipeline specifications.
- `methods/`: algorithm definitions.
- `experiments/`: benchmark recipes.
- `tasks/claude/`: research task cards.
- `tasks/cursor/`: code-building task cards.
- `prompts/`: ready-to-use LLM prompts.

Use the package as a blueprint, not as final truth.
Every claim must be tested.
